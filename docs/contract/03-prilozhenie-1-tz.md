# Приложение № 1  
## к Договору № ___ от «___» ______________ 2026 г.  
### Техническое задание (ТЗ) — интернет-магазин LETTO

**Продукт:** LETTO (цветочный магазин)  
**Сайт:** https://testletto.ru  
**Админка:** https://testletto.ru/admin  
**Репозиторий (ориентир):** https://github.com/MatveiShadchin/letto  

Настоящее ТЗ описывает объём результата, подлежащего передаче и приёмке.

---

### 1. Цель

Передать Заказчику работающий интернет-магазин цветов с админ-панелью, размещением на VPS с HTTPS и интеграцией Telegram (уведомления клиенту и переписка с флористами через группу).

---

### 2. Витрина (публичная часть)

2.1. Главная страница (в т.ч. товар дня, превью каталога).  
2.2. Каталог, карточки товаров, модальное окно «Подробнее».  
2.3. Корзина: изменение количества; аддоны (шары / игрушки / вазы).  
2.4. Вопрос об открытке к заказу (в корзине / чекауте).  
2.5. Блок рекомендаций в корзине («Дополните букет»).  
2.6. Оформление заказа (checkout): доставка и самовывоз.  
2.7. Самовывоз 24/7 (пункты); учёт сборки букетов по часам флористов — в объёме, реализованном на проде на дату сдачи.  
2.8. Отзывы, контакты.  
2.9. Адаптивная вёрстка (мобильные устройства и десктоп).

---

### 3. Админ-панель

3.1. Авторизация (вход).  
3.2. Дашборд.  
3.3. Товары: создание, редактирование, удаление; фото; категории; «товар дня» / popularity.  
3.4. Заказы и уведомления (в объёме, существующем на проде).

---

### 4. Telegram

4.1. Бот уведомлений клиенту по заказу (@letto_flowers_bot или актуальный username бота на дату сдачи).  
4.2. Привязка `chat_id` через `/start` и deep link вида `order_<uuid>`.  
4.3. Группа флористов: сообщение клиента боту → в группу; ответ флориста **реплаем** на сообщение бота → клиенту в боте.  
4.4. Допускается доставка сообщений через схему relay / sync (в т.ч. GitHub Actions) с задержкой порядка **до ~1 минуты** из‑за ограничений доступа к Telegram API с VPS в РФ.  
4.5. Мгновенный webhook через Cloudflare Worker (или иной proxy вне РФ) **не входит** в базовый объём, если не согласован отдельно в Приложении № 2.

**Требования к настройке на стороне Заказчика / совместной настройке:**  
- Privacy Mode бота в группе — Disable (BotFather);  
- бот добавлен в группу флористов (предпочтительно с правами администратора).

---

### 5. Инфраструктура

5.1. Стек: Next.js 13 (App Router), TypeScript, React, PostgreSQL, Nginx, PM2, Ubuntu VPS.  
5.2. Домен и HTTPS (Let’s Encrypt) для testletto.ru (или согласованного домена).  
5.3. Развёртывание на VPS; рабочий процесс деплоя в объёме проекта.  
5.4. Передача доступов — по списку в Акте.

---

### 6. Обучение

6.1. Инструкция и/или онлайн-обучение Заказчика работе с админкой — **1–2 часа**.

---

### 7. Не входит в объём работ (если не указано в Приложении № 2 отдельно)

7.1. Реклама, SMM.  
7.2. Наполнение каталога фотографиями, текстами и ценами (зона Заказчика).  
7.3. Юридическое лицо Заказчика, оферта, фискальные чеки.  
7.4. Онлайн-эквайринг (ЮKassa, СБП и аналоги).  
7.5. Интеграция с 1С, CRM, складом.  
7.6. Мобильное приложение (допускается использование сайта / PWA).  
7.7. Рабочие интеграции VK / WhatsApp / MAX (возможны заготовки API — не критерии приёмки).  
7.8. Доработки после подписания Акта, кроме гарантийных критических багов (раздел 7 Договора).

---

### 8. Известные ограничения (фиксируются как норма)

8.1. Задержка Telegram-сообщений до ~1 мин при текущей схеме доставки — допустима.  
8.2. Подрядчик не гарантирует бесперебойность Telegram API и сети третьих лиц.  
8.3. Оплата хостинга и домена — обязанность Заказчика (либо входит в платную поддержку — см. Приложение № 2).

---

### 9. Критерии приёмки (кратко)

Полный чеклист — в Акте. Минимум для приёмки:  
- сайт открывается по HTTPS;  
- каталог, корзина, оформление заказа работают;  
- админка: вход, товары, заказы;  
- сценарии Telegram клиент ↔ группа работают в смысле раздела 4;  
- доставка/самовывоз на форме отображаются корректно;  
- доступы переданы по списку.
